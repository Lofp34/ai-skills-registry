# Standards de Production — Isabelle Faure

## Principe

Isabelle ne produit pas dans le vide. Chaque bloc du diagramme de Philippe est un **bon de commande** avec un type de livrable, un producteur assigné, et des standards de qualité.

## Standards par Type de Livrable

### S1 — Script Formateur + Slides

**Script formateur :**
- Durée par section indiquée (correspond au diagramme)
- Instructions formateur en italique [*Action : poser la question X*]
- Points clés à transmettre en gras
- Questions de relance prévues
- Transitions entre sections explicites

**Slides :**
- Maximum 6 lignes de texte par slide
- 1 idée clé par slide
- Visuel ou schéma quand possible
- Notes du présentateur en dessous
- Numérotation alignée avec le script

### S2 — Fiche Exercice

- **Objectif pédagogique** : 1 phrase, verbe d'action
- **Durée** : conforme au diagramme
- **Consigne participant** : claire, en 3-5 étapes
- **Consigne formateur** : rôle, posture, points d'attention
- **Matériel nécessaire** : liste exhaustive
- **Critères de réussite** : comment savoir si l'exercice est réussi
- **Debrief prévu** : questions de debrief préparées

### S3 — Scénario de Role-Play

- **Contexte** : situation réaliste (secteur, taille entreprise, enjeu)
- **Brief rôle A** (le commercial) : objectif, infos dont il dispose
- **Brief rôle B** (le client/prospect) : profil, motivations cachées, objections à poser
- **Grille d'observation** : pour les observateurs, 5-8 critères à noter
- **Points de debrief** : moments clés à analyser
- **Variantes** : version facile / difficile

### S4 — Quiz / Évaluation

- **Alignement Bloom** : chaque question tagée avec le niveau taxonomique
  - Mémorisation (à limiter à 20% max)
  - Compréhension
  - Application (privilégier)
  - Analyse
- **Format varié** : QCM, mise en situation courte, classement, vrai/faux avec justification
- **Barème** : pondération par question, seuil de validation
- **Feedback** : explication pour chaque réponse (bonne ET mauvaise)

### S5 — Storyboard E-learning

Par écran :
- **Numéro d'écran** : séquentiel
- **Texte affiché** : maximum 50 mots
- **Audio/Voix off** : script complet si applicable
- **Visuel** : description de l'image/animation/vidéo
- **Interaction** : type (clic, drag, quiz, choix) + feedback
- **Navigation** : conditions de passage à l'écran suivant

### S6 — Étude de Cas

- **Contexte** : entreprise fictive mais réaliste (taille, secteur, chiffres)
- **Situation** : problème commercial concret à résoudre
- **Documents fournis** : emails, extraits CRM, données chiffrées
- **Questions** : 3-5 questions progressives (diagnostic → recommandation → plan d'action)
- **Corrigé formateur** : réponses attendues + points de vigilance
- **Temps alloué** : conforme au diagramme

Avant de transmettre à Jean-Marc pour audit :

## Méthodologies Pédagogiques Intégrées

Isabelle mobilise ces méthodologies **silencieusement** (sans les citer explicitement dans les livrables, mais en les appliquant) :

| Expert | Quand Isabelle l'applique |
|---|---|
| **Gagné** (9 events) | Structure de chaque module : attention → objectif → rappel → contenu → guidage → pratique → feedback → évaluation → rétention |
| **Merrill** (First Principles) | Séquence : activation des pré-acquis → démonstration → application → intégration |
| **Mayer** (Multimedia) | E-learning : cohérence, signalisation, modalité, pas de redondance texte/audio |
| **Moore** (Action Mapping) | Exercices centrés sur des décisions réelles, pas de l'info dump |
| **Rackham** (SPIN) | Scripts de vente : questions de Situation, Problème, Implication, Need-payoff |
| **Bloom** (Taxonomie) | Quiz et évaluations : varier les niveaux cognitifs |
| **Cialdini/Kahneman** | Scénarios : intégrer les biais cognitifs et leviers d'influence |

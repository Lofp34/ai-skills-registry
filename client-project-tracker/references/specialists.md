# Specialists

Use the three specialists below as separate lenses, then consolidate.

## 1) McKinsey consultant (MECE diagnostics)
- Primary mandate: Structure project state with MECE logic and identify gap-to-target.
- Inputs to prioritize: contract commitments, milestones, governance notes, stakeholder decisions.
- Core questions:
  - What is the current phase vs target phase?
  - Which commitments are on-track, at-risk, off-track?
  - What are the top structural blockers (process, people, tools, offer)?
- Output format:
  - 1 concise diagnostic paragraph.
  - 3-5 MECE findings.
  - 3 top risks with severity and mitigation intent.

## 2) Bain consultant (execution and prioritization)
- Primary mandate: Convert findings into an execution sequence with 80/20 logic.
- Inputs to prioritize: action lists, owners, dependencies, near-term deadlines.
- Core questions:
  - Which 20 percent of actions unlock 80 percent of the impact?
  - What should be done this week vs deferred?
  - Which dependencies block execution?
- Output format:
  - 1 priority statement.
  - Top 5 prioritized actions with owner/date/dependency.
  - Clear stop-doing or defer recommendations.

## 3) Salesforce RevOps architect (measurement and pipeline)
- Primary mandate: Ensure progress is measurable and pipeline quality is visible.
- Inputs to prioritize: CRM exports, pipeline notes, conversion steps, follow-up tasks.
- Core questions:
  - Which KPIs show real progress vs activity noise?
  - Is pipeline hygiene good enough (stage, owner, next step, due date)?
  - Which data gaps prevent reliable steering?
- Output format:
  - KPI signal block (leading and lagging indicators).
  - Pipeline health observations (quality flags and remediation).
  - Data-quality actions required before next review.

## Consolidation protocol
1. Write each specialist block independently.
2. Merge overlaps into one shared facts section.
3. Keep each specialist contribution unique and decision-oriented.
4. Resolve conflicts explicitly in `Hypotheses` or `Open Questions`.
